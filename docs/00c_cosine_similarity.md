# 보충 — 코사인 유사도 (`A_unit @ A_unit.T`) 깊이 보기

> 00b의 S5를 한 줄씩 분해한 문서.
> 헷갈리는 코드:
> ```python
> A_unit = A / np.linalg.norm(A, axis=1, keepdims=True)
> print(A_unit @ A_unit.T)
> ```
> 한 번에 보면 어렵다. **세 조각**으로 쪼개서 본다.

---

## 핵심 목적 (먼저 기억할 것)

> **왜 길이로 나누나? → 크기를 없애고 "방향"만 남기려고.**

`A @ A.T` 는 "크기까지 포함된 행끼리 닮음"이었다.
여기서 **크기를 빼버린 버전**이 코사인 유사도다. (= 방향만의 닮음)

학생 예시로: "총점은 무시하고, **어떤 과목에 치우쳤는지 그 패턴**만" 비교하는 것.

---

## 조각 1 — `np.linalg.norm(A, axis=1, keepdims=True)`

각 **행(샘플)의 길이**를 구한다.

```text
A[0]=[0,1,2,3]   → √(0+1+4+9)      = √14  ≈ 3.74
A[1]=[4,5,6,7]   → √(16+25+36+49)  = √126 ≈ 11.22
A[2]=[8,9,10,11] → √(64+81+100+121)= √366 ≈ 19.13
```

- `axis=1` → 행 방향으로 길이 계산 (axis 기억법: 1이 사라짐 → 행별 결과)
- `keepdims=True` → 결과를 `(3,1)` 모양으로 **세워둠** (이유는 조각 2)

결과:
```text
[[ 3.74],
 [11.22],
 [19.13]]     # shape (3,1)
```

---

## 조각 2 — `A / 길이` : 정규화 (단위벡터 만들기)

각 행을 **자기 길이로 나누면** 길이 1짜리 **단위벡터**가 된다.

```text
A[0] / 3.74  = [0, 0.27, 0.53, 0.80]
A[1] / 11.22 = [0.36, 0.45, 0.53, 0.62]
A[2] / 19.13 = [0.42, 0.47, 0.52, 0.57]
```

### `keepdims=True` 가 필요한 이유

- `A` 는 `(3,4)`, 길이는 `(3,1)` → 브로드캐스팅으로 **각 행을 그 행의 길이로** 나눔 ✅
- `keepdims=False` 면 길이가 `(3,)` → `(3,4) / (3,)` 는 모양이 안 맞아 **에러**
- → 직접 `keepdims` 빼고 돌려서 에러를 봐야 왜 필요한지 와닿는다.

`A_unit` 의 의미: 크기(점수 절대값)는 1로 통일, **방향(성적 패턴)만** 남긴 것.

---

## 조각 3 — `A_unit @ A_unit.T` : 내적이 곧 코사인

`A @ A.T` 는 "행끼리 내적 표"였다. 그런데 행이 **전부 길이 1**이면?

내적 공식:
$$
\vec{a}\cdot\vec{b} = |\vec a|\,|\vec b|\cos\theta
$$

길이가 둘 다 1이면 $|\vec a|=|\vec b|=1$ 이므로:
$$
\vec{a}\cdot\vec{b} = 1 \cdot 1 \cdot \cos\theta = \cos\theta
$$

> **길이를 1로 만들었더니 내적이 곧 $\cos\theta$ 가 됐다.**

그래서 `A_unit @ A_unit.T` = **행들 사이 사잇각의 코사인 표** = **코사인 유사도**.

```text
        학생0   학생1   학생2
학생0 [  1.00   0.83   0.79  ]
학생1 [  0.83   1.00   0.99  ]
학생2 [  0.79   0.99   1.00  ]
```

### 해석

- **대각선 = 1** : 자기 자신과 각도 0° → `cos0° = 1`. 항상 그렇다.
- **비대각선** : 1에 가까울수록 두 행의 **방향(패턴)이 비슷**.
- 학생1 ↔ 학생2 가 0.99 → "총점은 달라도 과목별 잘하는 패턴은 거의 동일".

---

## 전체 요약 (한 문장)

> 1. **길이로 나눠** 모든 행을 단위벡터로 (크기 제거, 방향만)
> 2. 그걸 **내적**하면 길이가 1이라 내적 = $\cos\theta$
> 3. 결과 = **코사인 유사도 표** = "크기를 빼고 방향만으로 본 행끼리 닮음 표"

| 표현 | 무엇을 보나 |
|------|------------|
| `A @ A.T` | 크기 + 방향 (닮음, 크기 영향 받음) |
| `A_unit @ A_unit.T` | **방향만** (코사인 유사도) |

---

## 과제 (노트북에서 직접)

```python
# 1. 정규화 후 각 행의 길이가 정말 1인지 확인
A_unit = A / np.linalg.norm(A, axis=1, keepdims=True)
print(np.linalg.norm(A_unit, axis=1))   # [1. 1. 1.] 나와야 함

# 2. keepdims=True 빼면 왜 안 되는지 — 일부러 에러내보기
print(A / np.linalg.norm(A, axis=1))    # 에러 (모양 안 맞음)

# 3. 코사인 표의 대각선이 전부 1인지
print(np.diag(A_unit @ A_unit.T))       # [1. 1. 1.]
```

- 1번에서 **전부 1**이 나오면 "진짜 단위벡터가 됐다"는 확인 → 이게 출발점.
